Introduction

This guide documents a production-tested implementation of fine-tuning TinyLlama-1.1B-Chat-v1.0 on the NVIDIA Jetson Orin Nano 8GB using LoRA (Low-Rank Adaptation) and Supervised Fine-Tuning (SFT). The implementation uses FP16 precision instead of 4-bit quantization because BitsAndBytes library doesn't support CUDA 12.6 on ARM64 architecture.

Actual Training Results (March 26, 2026):
  • Hardware: NVIDIA Jetson Orin Nano 8GB (CUDA 12.6, JetPack 6.2.1)
  • Training Time: 1331 seconds (~22.2 minutes) for 1 epoch on 3,000 samples
  • Speed: 1.77 seconds/iteration, 2.25 samples/second
  • Trainable Parameters: 563,200 out of 1,100,611,584 (0.0512%)
  • GPU Memory Usage: ~4.5GB during training
  • Initial Loss: 1.8216 (step 10)
  • Final Loss: 1.5203 (step 750)
  • Model Files: LoRA adapters (2.2MB), Merged model (2.2GB)
Critical Requirements for Jetson:
  • Close Cursor IDE: Cursor uses 2.5-3.5GB GPU memory → Must close before training
  • GPU Frequency: Manually set to 1020 MHz (jetson_clocks doesn't fully work on JetPack 6.x)
  • No BitsAndBytes: 4-bit quantization not supported on Jetson ARM64/CUDA 12.6 - use PyTorch native FP16
  • NumPy Version: Must use NumPy 1.x (not 2.x)
  • Transformers Version: Use 4.46.3 (not 4.47.x for TRL compatibility)

Step 1: Configuration (config.py)

Centralized configuration for all hyperparameters. This makes experimentation easy.

# File: config.py # Model Configuration MODEL_NAME = "TinyLlama/TinyLlama-1.1B-Chat-v1.0" MAX_MEMORY = { 0: "2.25GiB", "cpu": "16GiB" } # LoRA Hyperparameters ALPHA = 8 # LoRA scaling factor (alpha/r = 2.0) DROPOUT = 0.0 # No dropout for stability R = 4 # LoRA rank (lower = fewer params) BIAS = "none" # Don't train bias terms TASK_TYPE = "CAUSAL_LM" # Causal language modeling TARGET_MODULES = ["v_proj", "q_proj"] # Only attention query/value layers # Training Hyperparameters OUTPUT_DIR = "./results" PER_DEV_TRAIN_BATCH_SIZE = 1 # Physical batch size GRAD_ACC_STEP = 4 # Effective batch = 1 × 4 = 4 OPTIM_METHOD = "adamw_torch" # Standard AdamW optimizer LR = 2e-4 # Learning rate (typical for LoRA) LR_SCHED_TYPE = "cosine" # Cosine annealing schedule NUM_OF_EPOCHS = 1 # Single epoch for testing LOG_STEPS = 10 # Log every 10 steps FP16_BOOL_VALUE = True # FP16 mixed precision GRAD_CHECKPOINT_BOOL_VALUE = True # Gradient checkpointing (saves ~40% memory) NO_CUDA = False # SFT Configuration MAX_SEQ_LEN = 64 # Short sequences for memory efficiency
Key Configuration Choices:
  • r=4: Smallest rank for minimal memory (563k params). Use r=8 or r=16 for better quality if memory allows
  • alpha=8: Sets scaling to alpha/r = 2.0 (standard practice)
  • target_modules=[q_proj, v_proj]: Only attention layers. Could add k_proj, o_proj, gate_proj for more capacity
  • batch_size=1, grad_acc=4: Simulates batch of 4 without extra memory
  • max_seq_len=64: Short sequences reduce memory quadratically

Step 2: Data Preparation (sft_dataprep.py)

Load and format the UltraChat 200k dataset using chat templates for conversational format.

# File: sft_dataprep.py from datasets import load_dataset from transformers import AutoTokenizer def load_tokenizer_model(model_name): """Load the tokenizer from the model""" tokenizer = AutoTokenizer.from_pretrained(model_name, trust_remote_code=True) return tokenizer def define_dataset(dataset_name, batch_size=1, shuffle=True): """Create a dataloader from a dataset""" dataset = ( load_dataset(dataset_name, split="test_sft") .shuffle(seed=42) .select(range(3_000)) # Only 3k examples for fast training ) return dataset def format_prompt(example, template_tokenizer): """Format the prompt of the dataset""" chat = example["messages"] # apply_chat_template adds special tokens: <|user|>, <|assistant|> prompt = template_tokenizer.apply_chat_template(chat, tokenize=False) return {"text": prompt} def data_mapping(dataset, template_tokenizer): """Map the dataset to the format required for the model""" dataset = dataset.map( lambda example: format_prompt(example, template_tokenizer), batched=False, remove_columns=dataset.column_names # Remove original columns ) print(dataset["text"][0]) # Print first example return dataset class DataPrep: """Class to prepare the data for the model""" def __init__(self): print("Starting Data Preparation") self.tokenizer = load_tokenizer_model("TinyLlama/TinyLlama-1.1B-Chat-v1.0") self.dataset = define_dataset("HuggingFaceH4/ultrachat_200k") self.data_mapping = data_mapping(self.dataset, self.tokenizer) self.data_mapping.save_to_disk("data_mapping") if __name__ == "__main__": data_prep = DataPrep()

Key Concepts:
  • apply_chat_template: Automatically formats multi-turn conversations with special tokens
  • tokenize=False: Returns text strings (SFTTrainer will tokenize later)
  • remove_columns: Keeps only the "text" column for training
  • 3,000 samples: Sufficient for testing fine-tuning (~22 min training time)

Step 3: Model & Tokenizer Loading (sft_model_tokenization.py)

Load TinyLlama in FP16 precision. NO BitsAndBytes 4-bit quantization (not supported on Jetson ARM64/CUDA 12.6).

# File: sft_model_tokenization.py import torch from transformers import AutoModelForCausalLM, AutoTokenizer def model_select(model_name): """Load model in FP16 precision""" model = AutoModelForCausalLM.from_pretrained( model_name, torch_dtype=torch.float16, # FP16 precision (NOT 4-bit!) device_map="auto", # Automatic GPU placement low_cpu_mem_usage=True # Minimize CPU memory during load ) return model def tokenizer_select(model_name): """Load and configure tokenizer""" tokenizer = AutoTokenizer.from_pretrained( model_name, trust_remote_code=True ) # TinyLlama has no PAD token, use EOS as padding tokenizer.pad_token = tokenizer.eos_token tokenizer.padding_side = "right" # Pad on right for training return tokenizer class SFTModelTokenization: def __init__(self, model_name): print("Setting up Model for SFT") self.model = model_select(model_name) print(self.model.hf_device_map) # Show device placement: {'': 0} print("Setting up Tokenizer for SFT") self.tokenizer = tokenizer_select(model_name)

Step 4: LoRA Configuration (sft_lora_config.py)

Apply LoRA adapters to reduce trainable parameters from 1.1 billion to only 563,200 (0.05%)!

# File: sft_lora_config.py from peft import LoraConfig, get_peft_model def apply_lora(model, l_alpha, l_dropout, l_rank, l_bias, l_task_type, l_target_modules): print("Setting up lora config") peft_config = LoraConfig( lora_alpha=l_alpha, # 8 lora_dropout=l_dropout, # 0.0 r=l_rank, # 4 bias=l_bias, # "none" task_type=l_task_type, # "CAUSAL_LM" target_modules=l_target_modules # ["q_proj", "v_proj"] ) # NO prepare_model_for_kbit_training() - we're not using 4-bit! model = get_peft_model(model, peft_config) return model, peft_config

Step 5: Training Configuration (sft_trainer_config.py)

Setup training arguments and SFTTrainer with aggressive memory optimizations for Jetson.

# File: sft_trainer_config.py from trl import SFTTrainer from transformers import TrainingArguments def define_training_args(output_dir, per_dev_train_batch_size, grad_acc_step, optim_method, lr, lr_sched_type, num_of_epochs, log_steps, fp16_bool_value, grad_checkpoint_bool_value): training_arguments = TrainingArguments( output_dir=output_dir, # "./results" per_device_train_batch_size=per_dev_train_batch_size, # 1 gradient_accumulation_steps=grad_acc_step, # 4 optim=optim_method, # "adamw_torch" learning_rate=lr, # 2e-4 lr_scheduler_type=lr_sched_type, # "cosine" num_train_epochs=num_of_epochs, # 1 logging_steps=log_steps, # 10 fp16=fp16_bool_value, # True gradient_checkpointing=grad_checkpoint_bool_value # True ) return training_arguments def sft_trainer(model, dataset, tokenizer, training_args, max_seq_len, peft_config): trainer = SFTTrainer( model=model, train_dataset=dataset, dataset_text_field="text", # Column from data prep tokenizer=tokenizer, args=training_args, max_seq_length=max_seq_len # 64 tokens ) return trainer
Memory Optimization Settings:
  • batch_size=1: Minimum physical batch to fit in 8GB
  • gradient_accumulation=4: Simulates batch of 4 without extra memory
  • fp16=True: Mixed precision training (2x memory reduction, 2x speed)
  • gradient_checkpointing=True: Saves ~40% memory at cost of ~20% speed
  • max_seq_length=64: Short sequences (memory scales as O(n²))
Actual Training Performance:
  • Total steps: 750 (3000 samples ÷ 4 effective batch)
  • Speed: 1.77 seconds per iteration
  • Throughput: 2.25 samples/second
  • GPU memory: ~4.5GB during training
  • Total time: 1331 seconds (~22.2 minutes)

Step 6: Main Training Script (lora_finetune.py)

Orchestrate the complete training pipeline from data prep to model merging.

# File: lora_finetune.py import os os.environ["PYTORCH_CUDA_ALLOC_CONF"] = "expandable_segments:True" os.environ["TOKENIZERS_PARALLELISM"] = "false" import torch import config from sft_dataprep import DataPrep from sft_model_tokenization import SFTModelTokenization from sft_lora_config import apply_lora from sft_trainer_config import define_training_args, sft_trainer from sft_merge_model_weights import get_merged_model if __name__ == "__main__": # Step 1: Prepare data data_prep = DataPrep() dataset_transformed = data_prep.data_mapping # Step 2: Load model and tokenizer in FP16 sft_model_tokenization = SFTModelTokenization(model_name=config.MODEL_NAME) model_sft = sft_model_tokenization.model tokenizer_sft = sft_model_tokenization.tokenizer # Step 3: Apply LoRA model, peft_config = apply_lora( model_sft, config.ALPHA, config.DROPOUT, config.R, config.BIAS, config.TASK_TYPE, config.TARGET_MODULES ) model.enable_input_require_grads() model.print_trainable_parameters() # Step 4: Setup training training_arguments = define_training_args( config.OUTPUT_DIR, config.PER_DEV_TRAIN_BATCH_SIZE, config.GRAD_ACC_STEP, config.OPTIM_METHOD, config.LR, config.LR_SCHED_TYPE, config.NUM_OF_EPOCHS, config.LOG_STEPS, config.FP16_BOOL_VALUE, config.GRAD_CHECKPOINT_BOOL_VALUE ) # Step 5: Train print("Starting Training", flush=True) sft_trainer_obj = sft_trainer( model, dataset_transformed, tokenizer_sft, training_arguments, config.MAX_SEQ_LEN, peft_config ) train_result = sft_trainer_obj.train() print(train_result, flush=True) # Step 6: Save adapters print("Saving Model and Tokenizer", flush=True) sft_trainer_obj.model.save_pretrained("TinyLlama-1.1B-qlora") tokenizer_sft.save_pretrained("TinyLlama-1.1B-qlora") # Step 7: Merge adapters with base model print("Preparing the LoRA Model with merged weights", flush=True) merged_model_final = get_merged_model() merged_model_final.save_pretrained("TinyLlama-1.1B-merged")

Jetson Orin Nano Performance Tuning

Critical Issue: GPU Stuck at 612 MHz

jetson_clocks doesn't correctly set GPU max frequency on JetPack 6.x. You must manually force it to 1020 MHz!

Manual GPU Frequency Fix

# Check available frequencies: cat /sys/devices/platform/bus@0/17000000.gpu/devfreq/17000000.gpu/available_frequencies # Output: 306000000 408000000 510000000 612000000 714000000 816000000 918000000 1020000000 # Check current frequency (probably stuck at 612 MHz): cat /sys/devices/platform/bus@0/17000000.gpu/devfreq/17000000.gpu/cur_freq # Output: 612000000 # Force GPU to maximum frequency (1020 MHz): echo 1020000000 | sudo tee /sys/devices/platform/bus@0/17000000.gpu/devfreq/17000000.gpu/min_freq echo 1020000000 | sudo tee /sys/devices/platform/bus@0/17000000.gpu/devfreq/17000000.gpu/max_freq # Verify: cat /sys/devices/platform/bus@0/17000000.gpu/devfreq/17000000.gpu/cur_freq # Should now show: 1020000000 # Check with jtop: jtop # GPU frequency should show 1020 MHz (not 612 MHz)
Performance Impact:
  • At 612 MHz: 1.77 sec/iteration → ~22 minutes training
  • At 1020 MHz: 1.5-1.6 sec/iteration → ~18 minutes training
  • Speed increase: ~1.67x faster (15-20% improvement)

Key Learnings & Best Practices

✅ What Worked

  • FP16 instead of 4-bit: BitsAndBytes doesn't support Jetson ARM64/CUDA 12.6; PyTorch native FP16 is perfect
  • Small LoRA rank (r=4): Sufficient for fine-tuning, keeps memory low (563k params)
  • Gradient accumulation: Effective batch size of 4 with physical batch of 1
  • Short sequences (64 tokens): Dramatically reduces memory footprint
  • Gradient checkpointing: Essential for 8GB GPU memory
  • Closing Cursor IDE: Frees 2.5-3.5GB GPU memory for training
  • GPU at 1020 MHz: 15-20% faster training than 612 MHz
  • Modular code structure: Easy to debug and modify
  • TTY training: Frees an additional ~2GB by stopping GUI

❌ Common Pitfalls to Avoid

  • Don't use BitsAndBytes on Jetson: 4-bit quantization not supported on ARM64/CUDA 12.6
  • Don't forget to close Cursor: It uses 3GB+ GPU memory
  • Don't skip GPU frequency tuning: Massive performance impact (1.67x)
  • Don't use NumPy 2.x: Causes dependency conflicts with PyTorch
  • Don't use Transformers 4.47.x: Incompatible with TRL 0.12.2
  • Don't set max_seq_length too high: Memory usage scales quadratically
  • Don't train with GUI running: Loses 1.5-2GB GPU memory

Troubleshooting Guide

🔴 Error: Out of Memory (OOM)

Symptoms: 
RuntimeError: CUDA out of memory
NvMapMemAllocInternalTagged: ... error 12
NVML_SUCCESS == r INTERNAL ASSERT FAILED
Solutions:
  • Close Cursor IDE: Frees 2.5-3.5GB GPU memory
  • Stop GUI (run from TTY): Frees additional 1.5-2GB
  • Reduce batch size: Already at 1 (cannot go lower)
  • Reduce max_seq_length: Try 32 instead of 64
  • Reduce LoRA rank: Try r=2 instead of r=4

🟡 Error: BitsAndBytes 4-bit Quantization Not Supported

Symptoms: 
OSError: libcudart.so not found
ImportError: libbitsandbytes_cuda126.so
Solution:

Don't use BitsAndBytes on Jetson! The library doesn't support CUDA 12.6 on ARM64 architecture. Use FP16 instead:

# Instead of this (WRONG): quantization_config = BitsAndBytesConfig(load_in_4bit=True) # Use this (CORRECT): model = AutoModelForCausalLM.from_pretrained( model_name, torch_dtype=torch.float16, # FP16 precision device_map="auto" )

🟡 Error: NumPy Version Mismatch

Symptoms: 
A module that was compiled using NumPy 1.x cannot be run in NumPy 2.2.6
Solution:
pip uninstall numpy pip install "numpy<2.0" # Force NumPy 1.x

🟢 Issue: GPU Stuck at 612 MHz

Symptoms: 
jtop shows: GPU 612 MHz (should be 1020 MHz)
Training is 2-3x slower than expected
Solution:
# Manually set GPU to max frequency: echo 1020000000 | sudo tee /sys/devices/platform/bus@0/17000000.gpu/devfreq/17000000.gpu/min_freq echo 1020000000 | sudo tee /sys/devices/platform/bus@0/17000000.gpu/devfreq/17000000.gpu/max_freq

🟢 Issue: Model Not Loading

Symptoms: 
Model hangs or crashes during loading
GPU memory fills up but model doesn't load
Solution:
  • Close Cursor IDE first! It's using 3GB+ GPU memory
  • Run training from terminal, not from IDE
  • Check available GPU memory: nvidia-smi or jtop

🟢 Issue: Dataset Download Timeout

Symptoms: 
ReadTimeoutError: Read timed out.
Connection error during dataset download
Solution:
